# Environment template (no secrets)

Duplicate values from `.env.example` for human-readable docs. **Do not put real secrets in this file.**

> **Maintenance:** this table is **hand-synced** with [`.env.example`](.env.example); there is no generator. When you add, rename, or remove a variable in one file, update the other in the same change or the two will drift.

Baseline lane (Inspect provider auth — the only path for comparable runs):

| Variable | Example | Notes |
| --- | --- | --- |
| `ANTHROPIC_API_KEY` | `sk-ant-api-xxxxx` | Pay-per-use API |
| `OPENAI_API_KEY` | `sk-xxxxx` | Pay-per-use API |
| `ANTHROPIC_BASE_URL` | _(empty)_ | Optional gateway / proxy endpoint |
| `MOONSHOT_API_KEY` | `sk-moonshot-xxxxx` | Kimi K2.5 base-model baseline. Moonshot exposes an OpenAI-compatible endpoint; see `MOONSHOT_BASE_URL`. |
| `MOONSHOT_BASE_URL` | `https://api.moonshot.ai/v1` | OpenAI-compatible endpoint for Moonshot. Route via Inspect's OpenAI-compatible provider or a dedicated `moonshot/` model id — confirm exact Inspect provider routing during Phase 3 prep. |

Phase B live pilot (control plane on dev-box / VPS; full procedure in `docs/ops/dev-box-pilot.md`):

| Variable | Example | Notes |
| --- | --- | --- |
| `BYTELLM_API_KEY` | `sk-bytellm-...` | Preferred ByteLLM proxy key for Phase B; mapped to `ANTHROPIC_AUTH_TOKEN` / `OPENAI_API_KEY` in the pilot process |
| `BYTELLM_BASE_URL` | `http://127.0.0.1:4000` | ByteLLM router root; `OPENAI_BASE_URL` may instead point at the `/v1` URL |
| `BENCHEVAL_PILOT_MODEL` | `gpt-5.3-codex-2026-02-24` | Default model for matrix lanes |
| `BENCHEVAL_PILOT_CLAUDE_MODEL` | `gpt-5.3-codex-2026-02-24` | TB `claude-code` lane; use a ByteLLM `/v1/messages` supported id |
| `BENCHEVAL_PILOT_CODEX_MODEL` | `gpt-5.3-codex-2026-02-24` | TB `codex-cli` lane; use a ByteLLM `/v1/responses` supported id |
| `BENCHEVAL_ALLOW_PREFLIGHT_ONLY` | `1` | Exit 0 when only preflight blockers or BFCL waived (see runbook §6) |
| `BENCHEVAL_HARBOR_FORWARD_PROXY` | `1` | Forward host `HTTP_PROXY` into Harbor task env (matrix default) |
| `BENCHEVAL_ANTHROPIC_SYSTEM_ROLE_SHIM` | `1` | Local shim when router requires top-level `system` |
| `BENCHEVAL_SHIM_AUTH_TOKEN_ENV` | `BYTELLM_PROXY_API_KEY` | Host shim env var used for upstream auth injection; containers receive dummy runtime keys |
| `BENCHEVAL_HOME` | `/path/to/bundle` | Wheel-only config bundle root (`export-config-bundle.sh`) |
| `HTTP_PROXY` / `NO_PROXY` | _(host-specific)_ | Host egress; `verify_auth.sh` honors `ANTHROPIC_BASE_URL` / `OPENAI_BASE_URL` |

Experimental lane (Phase 4 only, not baseline):

- Claude Code gateway uses `ANTHROPIC_AUTH_TOKEN` / `apiKeyHelper` in the Claude Code config — not in this `.env`.
- Codex CLI standalone auth is handled by `codex` login, not env vars.
